Aqui está o seu README profissional, estruturado para ser o cartão de visitas do projeto, explicando as duas formas de execução e como configurar o Docker para acessar bancos de dados locais.🛡️ Aegis Anonymizer ProPrivacidade de dados levada a sério com IA e Engenharia de Dados.O Aegis Anonymizer é uma ferramenta robusta para a criação de bancos de dados de homologação. Ele mascara informações sensíveis (PII) através de uma abordagem híbrida que utiliza Processamento de Linguagem Natural (IA) e Motores de Regex.🚀 Como Usar o SoftwareVocê pode rodar o Aegis de duas formas. O Docker é o método recomendado, pois ele já configura todos os drivers de banco e modelos de IA automaticamente.Opção 1: Via Docker (Recomendado 🌟)Esta opção isola o ambiente e garante que todos os drivers (SQL Server, Postgres, etc.) funcionem sem conflitos no seu Windows ou Linux.Build da Imagem:Bashdocker build -t aegis-pro .
Execução:Bashdocker run -p 8501:8501 aegis-pro
Acesso: Abra no navegador: http://localhost:8501Opção 2: Instalação Manual (Python Local)Instale as dependências: pip install -r requirements.txtBaixe o modelo de IA: python -m spacy download pt_core_news_lgRode o software: streamlit run app.py🛠️ Guia de Preenchimento (Configuração de Rede)Ao usar o Docker, a comunicação com o banco de dados muda um pouco se ele estiver na sua própria máquina (Host).📍 Caso 1: O Banco está em um Servidor RemotoSe o seu banco (Postgres/SQL Server) está em um IP da rede (ex: 10.0.0.50 ou db.minhaempresa.gov.br), basta preencher o campo Host normalmente com este IP ou DNS.📍 Caso 2: O Banco está na sua máquina local (onde o Docker roda)O Docker enxerga "localhost" como o próprio container. Para acessar o banco que está instalado no seu Windows/Linux "do lado de fora" do container, use:Host no Windows/Mac: host.docker.internalHost no Linux: 172.17.0.1 (IP padrão da ponte Docker)⚙️ O que o software faz por baixo dos panos?Mapeamento Genético do Banco: O Aegis lê o esquema do banco, identifica Chaves Primárias (PK) e Estrangeiras (FK).Classificação Inteligente: Ele decide o que anonimizar:SKIP: IDs, datas e números que mantêm a integridade.SENSITIVE: CPFs, RGs e Nomes que precisam de substituição direta.TEXT: Logs e observações processados por IA (SpaCy) para remover nomes no meio de frases.Bypass de Integridade: Durante a carga, o software desabilita as travas de chaves estrangeiras, permitindo que os dados entrem em qualquer ordem sem erros.Estimativa por IA: O tempo restante é calculado com base no tempo real que o modelo de linguagem leva para processar cada linha, gerando um ETA preciso.📋 Requisitos de CampoCampoDescriçãoExemploHostEndereço do banco de dados10.5.1.20 ou host.docker.internalPortaPorta padrão do banco5432 (PG), 1433 (MSSQL), 3306 (MySQL)Banco OrigemBanco com dados reaisdb_producaoBanco DestinoBanco para onde os dados irão (deve existir)db_homologacaoChunk SizeQuantidade de linhas por lote1000 (Recomendado)⚠️ AtençãoNunca utilize o banco de produção como Destino.
